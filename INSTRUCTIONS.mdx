## Comprendre

Dans les sections précédentes, nous avons principalement fais des requete de lectures (Query et Select).

Nous allons voir maintenant comment Créer, Mettre à jour et supprimer des données.

**1 : Création de données**

```tsx
await db.insert(users).values({name: 'Andrew'})
// multiple row
await db.insert(users).values([{name: 'Andrew'}, {name: 'Dan'}])
```

📑 Le liens vers la doc https://orm.drizzle.team/docs/insert#insert-one-row

**2 : Mise à jour**

```tsx
await db.update(users).set({name: 'Mr. Dan'}).where(eq(users.name, 'Dan'))
```

📑 Le liens vers la doc https://orm.drizzle.team/docs/update

**3 : Suppression**

```tsx
await db.delete(users).where(eq(users.name, 'Dan'))
```

📑 Le liens vers la doc https://orm.drizzle.team/docs/delete

## Exercice

Dans cet exercice nous avons notre page d’administration de produits.

- Un tableau contenant la liste des produits
- Un formulaire permettant l’ajout et l’édition de produit

Le tout est déjà connecté aux `actions` comme nous l’avons fait des les modules précédents.

Nous allons ici nous concentrer sur les requetes dans actions.

- **🐶 Implémente les requêtes :**
  - `getCategories` : Pour remplir la combobox category
  - `getProducts(catId?: number)` : Tous les produits avec un filtre optionnel sur la `category`
  - `getProductByName(name: string)`, recherche de produit par son nom, il sera utilise pour éviter les doublons
  - `deleteProductDao` : La requête de suppression
  - `insertProductDao` : La requête d’insertion
  - `updateProductDao`: La requête de mise à jour

<aside>
💡 Nous les avons renommé Dao (Data Accès Object) pour ne pas les confondre avec les fonction de l’action

</aside>

```tsx
export const deleteProduct = async (product: Product) => {
  await deleteProductDao(product.id)
  revalidatePath('/exercises/crud')
}
```

Nous ne voulons pas mélanger des fonctions lié a la BDD et aux actions, c’est le principe **SRP (Single Responsibility Principle)** de clean architecture

PS : Pour simplifier l’inversion, mise à jour dans le formulaire nous avons créé une fonction `persist` qui se base sur l’id du produit

```tsx
export async function persistProductDao(product: Product) {
  await (product.id > 0 ? updateProductDao(product) : insertProductDao(product))
}
```

## Bonus

### 1. 🚀 Upsert

Précédemment nous avons gérer nous même le cas `Update` / `Insert` en faisant

```tsx
export async function persistProductDao(product: Product) {
  await (product.id > 0 ? updateProductDao(product) : insertProduct(product))
}
```

`Drizzle` permet de faire des `Upsert` (Update / Insert).

Doc : https://orm.drizzle.team/docs/insert#on-conflict-do-update

Pour fonctionner on utiliser un `insert` + `onConflictDoUpdate`

```tsx
await db
  .insert(users)
  .values({id: 1, name: 'Dan'})
  .onConflictDoUpdate({target: users.id, set: {name: 'John'}})
```

- **🐶 Adapte la fonction `persistProductDao` en utilisant cette technique**
